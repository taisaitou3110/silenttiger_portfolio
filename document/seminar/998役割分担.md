# Gemini CLI との共同開発における役割分担と注意点

このドキュメントは、Gemini CLI（AIアシスタント）と人間（あなた）が共同でソフトウェア開発を進める上での、役割分担、具体的な手順、そして注意点をまとめたものです。

---

## 1. 役割分担

プロジェクトにおける責任範囲を明確にします。

### **人間（あなた）の役割**

-   **意思決定者 (Decision Maker):**
    -   プロジェクトの最終的な目的、必要な機能、デザインなどを決定します。
    -   AIが提案する複数の技術や設計案の中から、採用するものを選択します。
-   **環境管理者 (Environment Manager):**
    -   自身のPCに開発ツール（Python, Node.jsなど）やライブラリ（boto3など）をインストールします。
    -   AWSやデータベースなどの外部サービスのアカウントを管理します。
-   **実行者・テスター (Executor & Tester):**
    -   AIが生成したコードやコマンドを、自身の環境で実行します。
    -   完成した機能が要件通りに動作するかをテストし、問題があれば報告します。
-   **認証情報管理者 (Credentials Manager):**
    -   **最も重要な役割**です。データベースのパスワードやAWSのアクセスキーといった機密情報を、自身のPC内でのみ安全に管理します。

### **Gemini CLI（AI）の役割**

-   **アシスタント (Assistant):**
    -   要件定義の際に質問を投げかけ、アイデアを具体化する手助けをします。
-   **設計者・プログラマー (Designer & Programmer):**
    -   要件に基づき、データベースのスキーマ、API仕様、画面レイアウトなどを設計し、提案します。
    -   設計に基づき、HTML, CSS, JavaScript, Python, SQLなど、あらゆる言語のコードを生成します。
    -   AWSを操作するための `boto3` スクリプトなど、具体的な指示書（コード）を作成します。
-   **デバッガー (Debugger):**
    -   報告されたエラーメッセージやバグに基づき、問題箇所を特定し、修正案を提示します。

---

## 2. 具体的な開発の手順

一般的な開発プロセスは以下の通りです。

1.  **要件定義:** あなたが「何を作りたいか」を提示し、AIが質問で深掘りして要件を固めます。
2.  **設計:** AIがデータベース構造や使用技術を提案し、あなたが承認します。
3.  **環境構築:** あなたが、決定した技術に必要なツール（例: `pip install boto3`）をPCにインストールします。
4.  **実装（反復）:** 以下のサイクルを繰り返します。
    1.  **AI:** 具体的な機能を実現するためのコード（例: `app.py`）を作成・提案します。
    2.  **あなた:** そのコードをPC上で実行します（`run_shell_command "python app.py"`など）。
    3.  **AI:** 実行結果（成功、エラーなど）を確認し、次のコードを提案、または修正します。
5.  **テスト:** あなたがシステム全体を実際に操作し、意図通りに動くかを確認します。バグがあればAIに報告します。
6.  **デプロイ:**（必要であれば）AIがAWSなどにアプリケーションを配置するためのスクリプトを作成し、あなたが実行します。

---

## 3. 注意点（特にセキュリティとコスト）

### **セキュリティについて**

-   **最重要:** **APIキー、シークレットキー、パスワードなどの機密情報を、絶対にプロンプトに直接入力しないでください。**
-   **正しい管理方法:** 認証情報は、環境変数や、Gitで管理しないローカル設定ファイル（例: `~/.aws/credentials`）に保存してください。AIが作成するスクリプトは、それらの設定を自動的に読み込んで利用するようにします。
-   **基本原則:** AIは「金庫の鍵」を直接預かる**金庫番**ではありません。あなたに「この鍵で金庫を開けて」とお願いする**指示役**です。

### **コストについて**

-   **AIはコストを把握できない:** AIはあなたのGemini APIやAWSの現在の利用料金、無料枠の残量などを知ることはできません。
-   **コスト管理は自己責任:** クラウドサービスの利用料金は、あなた自身の責任で管理する必要があります。
-   **定期的な確認:** 意図しない高額請求を避けるため、**Google Cloud Console の請求ダッシュボード**やAWSの請求画面を定期的に確認する習慣をつけてください。
-   **課金体系:** 利用料金は、基本的にAIとの対話量（トークン数）や、実行したツールの処理量に応じた**従量課金制**です。
